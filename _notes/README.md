# 🪨 _notes — Obsidian 작업 공간

여기는 **아이디어와 주제를 자유롭게 메모**하는 곳이다. 언더스코어(`_`) 폴더라
**블로그에는 발행되지 않는다.** 마음껏 초안을 쓰고, 준비되면 글로 발행한다.

## 폴더
```
_notes/
├── README.md            ← 지금 이 문서
├── templates/
│   └── 퍼즐-글-템플릿.md   ← 새 글 노트 만들 때 복사해서 시작
└── topics/
    ├── 예시-주제.md        ← 작성 예시 (지우거나 참고)
    └── (내 주제 노트들)
```

## Obsidian 최초 설정 (1회)
1. Obsidian → **Open folder as vault** → 이 저장소 폴더 선택.
2. 왼쪽 파일 목록에서 `_notes/` 를 연다.
3. (선택) **Git 플러그인** 설치 → 메모가 자동 커밋·푸시된다.
   설정: Community plugins → Obsidian Git → Vault backup interval 10분 등.
4. (선택) **Templates 코어 플러그인** 켜고 템플릿 폴더를 `_notes/templates` 로 지정하면
   새 노트에 템플릿을 한 번에 삽입할 수 있다.

## 평소 흐름
1. `topics/` 에 새 노트를 만든다 (템플릿 복사).
2. 생각나는 대로 5칸(질문/조각1~3/가져갈 것)을 채운다. 문장 아니어도 된다.
3. 관련 주제는 `[[다른 노트]]` **위키링크**로 잇는다 → 그래프 뷰에서 연결이 보인다.
4. 준비되면 에이전트에게 발행을 맡긴다 (아래 프롬프트).

## 발행 프롬프트 (복붙)
```
_notes/topics/(노트파일) 을 블로그 글로 발행해줘.
- _branding/00-history-source.md 를 사실 근거로, 01·07 규칙을 따른다
- 07 템플릿의 퍼즐 형식 (질문 → 🧩 조각 → 완성된 그림 → 🎁 가져갈 것)
- 전문 용어는 💡 개념 카드 + 실재하는 레퍼런스
- [[위키링크]]는 대상 글이 있으면 실제 링크로, 없으면 일반 텍스트로
- 히어로 이미지도 만들어 assets/img/posts/ 에 넣어줘
- _posts/ 에 초안 생성 → 나는 검수만
```

## 위키링크 → 블로그 링크 규칙 (에이전트가 처리)
- `[[수천 명의 출전표에서 우리 아이 찾기]]` → 그 제목의 발행 글이 있으면
  `/posts/수천-명의-출전표에서-우리-아이-찾기/` 링크로 변환.
- 대상 글이 아직 없으면 링크를 걸지 않고 일반 텍스트로 둔다 (깨진 링크 방지).
- 그래서 노트에서는 마음껏 `[[ ]]` 로 이어도 된다 — 발행할 때 정리된다.

## 팁
- **그래프 뷰**(왼쪽 사이드바 그래프 아이콘)로 주제 연결을 보면, 사이트의
  [탐구 지도](https://kyj5482.github.io/#history)처럼 "나를 중심으로 뭐가 연결되는지"가 보인다.
- 노트 하나가 꼭 글 하나일 필요는 없다. 큰 주제 노트 아래 여러 글 아이디어를 담고,
  발행할 때 쪼개도 된다.
- 이미 발행한 글도 여기서 관리하고 싶으면, `_posts/` 를 Obsidian에서 함께 열어 참고한다.
