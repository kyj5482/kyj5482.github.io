# 05. 본인 준비 체크리스트 — 에이전트가 대신할 수 없는 것들

> 이 목록의 항목들은 결정권·계정 소유·법적 확인이 필요해서 **본인만 할 수 있다.**
> 완료하면 체크하고 결정 내용을 메모로 남긴다. 결정 내용이 곧 에이전트의 다음 작업 지시가 된다.

## 즉시 (Phase 0, ~2026.09)

- [ ] **회사 규정 확인 (가장 중요)**: 주재원 신분으로 ① 개인 블로그/뉴스레터 운영 ② 앱 출시 ③ 수익 발생이 겸업 금지·보안 규정에 저촉되는지 인사팀/법무 확인. 수익화 시점(Phase 3)을 조정해야 할 수 있음 → 결과에 따라 02 로드맵 수정
- [ ] **가족 회의**: 아이들 콘텐츠 노출 범위 합의 (익명/실명, 얼굴, 기록 공개 여부). 아이들 의사가 최우선 → 결과를 01 문서 가드레일에 반영
- [ ] **프로필 사진 촬영**: 블로그·LinkedIn·뉴스레터 공용 1장 (자연광, 상반신, 밝은 배경 권장)
- [ ] **LinkedIn 헤드라인 결정**: 에이전트가 초안 3개 제시 → 본인 선택
  - 예시 A: `Platform Engineer (20+ yrs, Samsung→Hyundai→Kia) | Building with AI Agents | Solo Startup in Public`
  - 예시 B: `Vehicle Data Platform @ Kia US | 20 yrs building 0→1 platforms | AI-native solo builder`
- [ ] **뉴스레터 플랫폼 선택**: 한국어 독자 중심이면 스티비 또는 Maily, 글로벌 확장 고려하면 Substack. 권장: **Maily(무료, 한국어 친화)로 시작 → 500명 넘으면 재평가**
- [ ] **X(트위터) 유지 여부 결정**: _config.yml에 계정 연결돼 있음. 운영 안 할 거면 링크 정리

## Phase 1 진입 전 (~2026.12)

- [ ] **YouTube 채널 개설** (쇼츠용): 채널명 결정 — B트랙 중심이므로 수영 관련 이름 권장 (앱 이름과 통일하면 최적)
- [ ] **수영 앱 이름·도메인 결정**: 에이전트가 후보 10개 + 도메인/상표 가용성 조사 → 본인 선택
- [ ] **개인 도메인 구매 여부**: kyj5482.github.io 유지 vs 커스텀 도메인 (예: 이름.dev). 브랜딩상 커스텀 권장, 급하지 않음
- [ ] **Google Analytics 연결**: _config.yml의 `google_analytics.id` 비어 있음 — GA4 생성 후 ID만 주면 에이전트가 반영
- [ ] **아이들 기록 데이터 정리**: 앱 MVP의 시드 데이터가 됨 (대회일, 종목, 기록, 장소). 형식은 자유 — 에이전트가 구조화

## Phase 2~3 진입 전 (2027)

- [ ] **Apple/Google 개발자 계정** 등록 (앱 출시용, 법인 vs 개인 명의 결정 — 회사 규정 확인 결과와 연동)
- [ ] **사업자 등록 여부 판단**: 매출 발생 전 세무 구조 결정 (미국 거주 + 한국 매출 조합은 세무사 상담 필요)
- [ ] **수영 커뮤니티 접점 만들기**: 한국 수영 학부모 카페, 미국 팀 커뮤니티 — 베타 테스터 모집 채널

## 매주 반복 (본인의 유일한 정기 업무)

- [ ] 주말: 주제 선택 + 음성메모 입력 (30분)
- [ ] 월요일: 초안 검수 + 발행 승인 (30분)
- [ ] 발행 전 체크 4항목 (03 문서 5절) — 이건 절대 에이전트에게 위임하지 않는다
