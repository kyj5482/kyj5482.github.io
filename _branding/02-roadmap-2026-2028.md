# 02. 로드맵 2026 H2 → 2028 — 마일스톤

> 원칙: 목표는 **통제 가능한 지표(발행 횟수)** 를 우선하고, 결과 지표(구독자·다운로드)는 후행 지표로만 본다.
> 처음 6개월의 유일한 목표는 "루틴이 끊기지 않는 것"이다. 조회수가 아니다.

## 전체 그림

```
2026 H2          2027 H1           2027 H2           2028
[기반 구축]  →  [리듬 + 앱 베타] → [수익화 실험]  →  [확산]
블로그 재가동    뉴스레터 런칭      앱 정식 출시       브랜드 자산 통합
LinkedIn 정비    쇼츠 채널 시작     스폰서/멘토링      영문 콘텐츠 확대
콘텐츠 시스템    앱 MVP 개발        구독자 성장        앱 성장/수익
```

## Phase 0 — 기반 구축 (2026.07 ~ 2026.09)

이번 분기의 목표는 단 하나: **시스템이 돌아가기 시작하는 것.**

| # | 마일스톤 | 완료 기준 | 담당 |
|---|---------|----------|------|
| 0-1 | 블로그 리브랜딩 | About 갱신, 선언문 포스트 발행, 카테고리 구조 확정 | 에이전트 (이번 작업) |
| 0-2 | 콘텐츠 시스템 가동 | 03 문서의 주간 루틴 4주 연속 실행 | 본인 + 에이전트 |
| 0-3 | LinkedIn 프로필 정비 | 헤드라인/About/경력 재작성 (05 체크리스트 참고) | 본인 결정 + 에이전트 초안 |
| 0-4 | 글 8편 발행 | 주 1편 × 8주 (A트랙 5 + B트랙 3) | 에이전트 초안 + 본인 검수 |
| 0-5 | 수영 앱 컨셉 확정 | 문제 정의 1페이지 + 핵심 기능 3개 + 이름 후보 | 본인 + 에이전트 |
| 0-6 | 회사 겸업/보안 규정 확인 | 주재원 신분에서 앱 출시·수익화 가능 범위 확인 | **본인만 가능** |

## Phase 1 — 리듬 + 앱 MVP (2026.10 ~ 2027.03)

| # | 마일스톤 | 완료 기준 |
|---|---------|----------|
| 1-1 | 발행 리듬 상향 | 주 1 블로그 + 주 1 LinkedIn 재가공 + 격주 B트랙 글 |
| 1-2 | 뉴스레터 런칭 | 플랫폼 선정(05 참고) → 월 2회 발행 시작, 첫 100명 |
| 1-3 | 쇼츠 파일럿 | B트랙 쇼츠 10편 제작·게시 → 반응 데이터로 포맷 확정 |
| 1-4 | 수영 앱 MVP | 가족이 실제로 쓰는 버전 (대회 기록/훈련 로그 등 핵심 1기능) |
| 1-5 | Build in Public 시리즈 | 앱 개발 과정을 블로그+LinkedIn에 월 1회 공개 |

## Phase 2 — 앱 베타 + 채널 정착 (2027.04 ~ 2027.09)

| # | 마일스톤 | 완료 기준 |
|---|---------|----------|
| 2-1 | 앱 베타 공개 | 수영 선수 학부모 커뮤니티 대상 비공개 베타 → 피드백 루프 |
| 2-2 | 쇼츠 정례화 | 주 2편, 확정된 포맷으로. 앱 대기자 명단 유입 연결 |
| 2-3 | 뉴스레터 500명 | 블로그/쇼츠 → 뉴스레터 전환 퍼널 점검 |
| 2-4 | LinkedIn 영문 확대 | 월 2회 영문 포스트 (미국 시장 = 앱의 두 번째 시장) |
| 2-5 | 첫 외부 노출 | 팟캐스트 출연, 커뮤니티 발표, 기고 중 1건 이상 |

## Phase 3 — 출시 + 수익화 실험 (2027.10 ~ 2028.03)

| # | 마일스톤 | 완료 기준 |
|---|---------|----------|
| 3-1 | 앱 정식 출시 | 앱스토어 출시 (한국 → 미국), 첫 유료 전환 실험 |
| 3-2 | 수익 실험 3종 | 앱 구독/광고, 뉴스레터 스폰서, 멘토링·강연 중 실측 |
| 3-3 | 첫 매출 | 금액보다 "0 → 1" 달성 자체가 마일스톤 |

## Phase 4 — 확산 (2028)

- 브랜드 자산 통합: 블로그·뉴스레터·쇼츠·앱이 하나의 퍼널로 동작
- 영문 콘텐츠 비중 확대 (LinkedIn + 앱 미국 시장)
- 선택지 열어두기: 책/강의, 앱 라인업 확장, 본격 창업 전환 여부 판단
- 아이들이 원할 경우 본인 채널로 독립할 수 있는 기반 마련

## 분기 점검 (Quarterly Review)

매 분기 마지막 주에 에이전트와 함께 점검한다:
1. 발행 목표 대비 실적 (몇 주를 건너뛰었나, 왜)
2. 후행 지표 추이 (구독자, 조회, 다운로드) — 판단 재료일 뿐 목표 아님
3. 다음 분기에 버릴 것 1개, 더할 것 1개
4. 이 문서와 01 전략 문서 갱신
