# 퍼스널 브랜딩 운영 문서 (_branding)

이 폴더는 Jekyll이 렌더링하지 않는 **작업 문서**입니다. (언더스코어 폴더는 사이트에 배포되지 않음)
퍼스널 브랜딩의 전략–로드맵–실행 시스템을 여기서 관리하고, 실제 산출물(글, 쇼츠 스크립트, 뉴스레터)은
Claude Code 에이전트에게 이 문서들을 컨텍스트로 주고 생성합니다.

## 문서 구성

| 문서 | 내용 | 갱신 주기 |
|------|------|----------|
| `01-brand-strategy.md` | 포지셔닝, 두 개의 브랜드 기둥, 메시지 하우스 | 분기 1회 점검 |
| `02-roadmap-2026-2028.md` | 2026 H2 → 2028 마일스톤과 단계별 목표/지표 | 분기 1회 점검 |
| `03-content-system.md` | AI 에이전트 기반 콘텐츠 생산 파이프라인, 주간 루틴, 프롬프트 템플릿 | 필요 시 |
| `04-topic-backlog.md` | 글/쇼츠/뉴스레터 주제 백로그 (두 트랙) | 매주 추가 |
| `05-checklist.md` | 본인이 직접 결정·준비해야 하는 항목 (에이전트가 대신 못 하는 것) | 완료 시 체크 |

## 에이전트 활용 방법

새 콘텐츠를 만들 때 Claude Code에 이렇게 요청합니다:

```
_branding/01-brand-strategy.md 와 03-content-system.md 를 읽고,
04-topic-backlog.md 의 [주제 번호] 주제로 블로그 글 초안을 작성해줘.
내 생각: (여기에 음성메모/메모를 붙여넣기)
```

원칙: **생각과 경험은 내가 제공하고, 구조화와 완성은 에이전트가 한다.**
에이전트가 만든 초안은 반드시 본인이 15–30분 검수 후 발행한다. (사실관계, 회사 보안, 가족 프라이버시)
