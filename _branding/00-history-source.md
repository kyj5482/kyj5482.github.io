- 현대차/기아/제네시스 브랜드 고객이 하나의 어카운트로 글로벌 서비스 활용하도록 설계·개발
- 현대차에서 1인 개발자로 시작, 1차 개발 완료 시점에 2명 추가 → 스타트업처럼 일하며 대기업 백만 이상 트래픽 경험

- 당시 본인은 클라우드 서비스 개발을 PoC 수준 + 삼성 어카운트 API 연동 경험이 전부
- 새벽 3시 출근 개발, 오전 9시부터 회의

**커넥티드카 서비스 플랫폼 (2017~2018):**
- 유럽 시작으로 글로벌 런칭 필요. 기존 한국·미국·중국이 완전 분리 개발됨
- 서버쪽 공식 문서 거의 없음 (오토에버 통해 2차 협력사가 개발, 인도·중국 등 파편화 → 일원화 난제)
- 비즈니스 관점: 나라마다 신규 런칭/업데이트에 막대한 비용, 앱 개발까지 고려하면 10억 이상
- 본인 포함 3명이 유럽→한국→중국→인도까지 2018년까지 차례로 런칭 + 직접 운영
- 설계·PoC: 경력 입사한 훌륭한 팀원과 시작. 차량 연결이라 보안성 우수 + 플랫폼 기반 일원화 + 원앱 글로벌 런칭 목표
- 서버 언어: 성능 위주 서비스는 Go, 데이터 처리 많은 것은 Node.js
- 보안: 계정 OAuth + 파트너사마다 고객 데이터/차량 기능 인증·인가용 JWT. 온프레미스 DMZ존 실행 API Gateway로 tyk 선정
- 오픈소스만 활용, API 추가 시 CI/CD에서 tyk 자동 반영 → 운영 편의성
- 앱 API 호출 보안: 엠엔소프트(현 오토에버)에 가이드했으나 비용·기간 부족으로 대부분 웹뷰. 모바일 맞춤형 웹도 직접 개발·배포
- 자동차 평가 특성상 모바일 네트워크 음영지역 테스트, 엣지케이스 지속 업데이트
- 최종 구성: API G/W, Account, Vehicle(차량 기준 데이터·기능 제어), Partner(홈투카·카투홈 등 파트너 연계 Developer Portal·계정 발급), Smart Phone Application 서버 — 단 3명으로 완료, 유럽 런칭·운영
- 브랜드 3개 = 사실상 회사 3개, 나눠 진행 불가. 각 회사가 원하는 디자인·프로세스 모두 대응, 새 차량마다 세계 곳곳에서 직접 버그 수정·패치
- 백업 플랜 없음 → 실패 시 차량 출시 문제. 해외 해킹 시도 소식·타사 계정 데이터 해킹 뉴스마다 심장 멎는 스트레스. 하지만 단단해진 소중한 경험
- 성과를 개발자들이 갖도록 못 만들어줌 → 진급 등 문제로 팀원들 SKT·Naver 이직. 본인은 팀장/TF장이 아니어서, 그들이 좋은 조건으로 가도록 훌륭한 인재임을 설명하는 게 최선이었음

**Vehicle Twin (데이터 부문, 2년간 실리콘밸리 6개월):**
- 이직 시 리스트업의 마지막이 데이터 부문
- 차량은 엄청난 데이터 발생, 테슬라는 주행 데이터 수집·학습 중이나 현대차는 개발 당시 정의된 데이터만 수집 + 10년째 제대로 활용 못하는 빅데이터플랫폼
- 근본 변경 위해 차량 내부 제어기와 서버가 완전 결합하는 디지털 트윈 필요
- 가족을 한국에 두고 2년에 걸쳐 6개월간 미국 실리콘밸리 스타트업에서 디지털 트윈 설계 + 개발 스펙 정의
- 범위가 넓어 이름을 Vehicle Twin으로 명명
- 서버가 통신하는 대상 제어기는 차량 내 모든 제어기와 통신 가능, 수집 정책·프로토콜을 잘 정의하면 향후 업데이트 없이도 구축 가능하게 설계 (과거엔 수정 필요 시 차량 개발 프로세스상 최소 3년 소요)
- 삼성에서 스터디한 오프로딩 기술 적용하려 했으나 차량 단말 제어기의 프로세싱·메모리 부족 → To-Do로 남김. 차량-서버 스펙 완료 후 첫 양산까지 서버 개발 집중
- 주요 서비스: 원격진단서비스. 차량 문제 해결에 늘 서비스 방문 필요 → 고객 불만 증가, 간헐적 문제는 확인 안 됨. Vehicle Twin으로 해결 가능 예상
- 결과: 새 차량은 서비스가 원하는 방식·데이터를 비용 최적화로 수집, 차량 데이터의 즉시 활용 불가 문제를 Vehicle Twin 서버가 서비스 친화적으로 제공해 즉시 활용 수준으로 향상
- Top-Tier 진입 위해선 수집된 차량·비즈니스 데이터를 종합 학습·모델링해 지속 발전해야 하나, 기존 수집 담당 부서 프로세스가 기존 방식에 신규를 편입하며 발전 정체
- Vehicle Twin 서버는 네이밍으로 여러 곳에서 공격받아 Vehicle Data Stream으로 개명해 운영

### 미국 기아 주재원 (2024~)
- Vehicle Twin 완성의 마지막 퍼즐: 비즈니스에 차량 데이터를 직접 연결해 회사 이익에 기여하고자 미국 기아 주재원
- 미국은 최대 판매량·이익, 기아 북미권역(미국·캐나다·멕시코 판매법인 + 조지아 공장) 담당 → 목표에 최적
- 빅데이터센터: 100여 대 물리 서버로 자원 충분하나 활용은 10대 이하, 대부분 스토리지로만 사용
- 개선: 운영 안정성 향상 + 활용 가능 수준 파이프라인 구축
- 연 2M 달러+ 수집 비용 발생. 차량 판매 증가 + 수집 정책 변경(1년→3년)으로 문제 확대 예상
- 매일 1M 차량 수집하나 활용은 하루 1대도 분석 안 되는 미스매치 → 안전 분석 필수 데이터 정의·축소 진행. 담당자와 많은 논쟁, 향후 분석 위한 수집 추가·활용 가속 필요성 어필해 적용
- 기존 이메일 기반 데이터 요청·수동 파싱·제공을 API 개발로 24/7 지원
- 소스부터 수집·처리해 데이터마트 구축. 과거엔 리소스 부족으로 공통화·표준화 마트 설계 집중, 지금은 스토리지 저렴 → 목적성 기반 마트, 마스터 데이터만 잘 구성하고 서비스·의사결정 포인트별 마트 구성 → 활용성 증가
- 안전 측면 Ondemand 분석하던 차량 데이터를 법인이 Feature 선정하면 기본 통계(Min/Max/P10/P50/P90 등) 일·월 Summary 확보
- 결과: 고객 불만 최상위 품질 이슈 중 하나인 12V 지속 모니터링 → 암전류 발생 즉시 판단 가능. 비즈니스 데이터와 결합해 S/W 업그레이드 수리 후에도 현상 재현 시 수리 방법 문제를 알려주도록 업그레이드. 유스케이스를 점차 확대하는 Vehicle Twin 플랫폼 구축

### 2026~ AI 에이전트
- AI 기술 확대로 데이터 기반 AI 에이전트 증가
- 서비스 배포: 계정·메뉴·접근권한 등 공통은 ai1.admin, 판매(ai1.sales)·마케팅(ai1.marketing)·서비스(ai1.service)·상품(ai1.product)·품질(ai1.quality)로 부문별 마이크로앱 지속 출시
- 2주 스프린트 타겟이나 대부분 1일 1앱 출시 가능 → 유스케이스 최대 발굴, Quick Win(2주) 후 평가해 Pilot·Scale로 발전하는 프로세스 구축 중
- 향후: 북미권역 데이터 앱마켓 개발, 누구나 정의한 데이터가 파이프라인으로 마트 제공되고 그 위에서 앱 동작. 2027년 7월경 데이터 앱스토어 출시 → 누구나 앱 개발·배포 가능
- ※ 이 부분은 미출시 계획이므로 콘텐츠화 시 공개 범위 주의 (01 가드레일)

## 개인 히스토리

### 운동 이력
- 초등: 축구 / 중등: 농구·볼링 / 고등: 검도·스쿼시, 학교 대회 수준 참여. 운동을 좋아하고 친구들과 늘 함께할 실력
- 거제도(한국 두 번째로 큰 섬) 출생. 수영은 대학 때 서울에서 배움 → 재미를 느껴 삼성 입사 후에도 지속
- 대학 때 골프·스포츠댄스 교양, 겨울마다 스노보드. 대학원 때 스키 재미 → 스키도. 오래 하나만 하지는 못함

### 아이들 수영
- 본인 경험으로 아들·딸과 어릴 때부터 공차기·인라인·물놀이·등산·자전거 함께
- 유치원 선택 시 영어유치원 대신 유아체능단 → 처음 수영 배움, 1년 만에 동네 대회 입상
- 아들: 초등 입학하며 코로나로 수영 잠시 중단 → 강남구 센터에서 재개
- 딸: 당시 거의 수영 못하는 수준이었으나 물을 좋아해 발차기부터 다시 적응. 폼이 안 좋아 1~2달 프라이빗 레슨으로 교정, 점차 마스터즈반 향해 나아감
- 개포 거주 → 과천 이사. 마스터즈반 있는 가까운 곳 문의했으나 코로나로 선수 등록해야만 가능
- 심사숙고 후 아들은 선수 등록 + 마스터즈반, 딸은 레슨반에서 더 배움
- 2달도 안 되어 전국소년체전 경기도대표 선발전 참가 (공식 첫 대회) → 9위, 결승 진출 실패로 속상
- 이 경험 계기로 당시 3학년 대회 대부분 8위로라도 결승 진출해 경험 축적
- 주종목 평영 외에 접영으로 전국대회 출전 → 3위 입상, 첫 큰 성과
- 딸: 마스터즈반으로 옮겨 대회 출전, 생활체육 부문 경기도대표에 접영·평영으로 선발, 실력·경험 축적

### 대회 원정 = 여행 철학
- 아이들과 전국 대회 참가. 처음엔 대회만 다녀오고 다른 것 못 함
- 아이들이 커서 대회 기억밖에 없을까 봐 그 지역 유명 먹거리·볼거리를 하나 이상 꼭 경험시킴
- 지역마다 뭐가 맛있고 어디가 좋은지 알게 됨 → 대회가 기록만이 아니라 지역 경험 선물이 됨. 지금도 어디서 뭐 먹고 싶다는 대화 주제
- 아이 키우며 운동 함께하다 보니 친구 만남 감소, 또래보다 빠른 육아로 타이밍 안 맞아 더 가족적 마인드·생활 습관으로 변화
- 개인 사업 하고 싶었으나 경험 없고 아이들과 함께가 무엇과도 비교 못할 만큼 좋아 사업은 미룸

### 여행·해외 지향
- 어릴 때부터 영어 못했으나 군 전역·비트교육센터 수료 후 혼자 미국 여행 계획, 20일간 다녀옴
- 2006년이라 아이폰·지도 없이 사람들에게 물어 대중교통으로 LA·라스베거스·샌프란시스코·샌디에이고
- 보스턴·뉴욕 가서 대학원 더 보고 싶었으나 국내선 티켓 너무 비싸 포기
- 이 혼자 여행 경험이 이후 여행 습관과 세상 보는 눈이 달라진 첫 계기
- 이후 밴쿠버 학교 지원 2개월 문화탐방 → 해외 거주 욕구 커짐 (당시 밴쿠버는 무인 스카이트레인 운행, 미국·캐나다는 기술적으로 발전)
- 출장으로 많은 나라 경험했으나 해외 거주 갈망은 여전
- 해외 거주엔 영주권 필요 + 개인 사업 욕구 → 스타트업 비자로 제2의 삶 계획했으나, 회사 주재원으로 미국 서부行 가능해 영주권 포기하고 미국 입국
- 미국은 여전히 크고 강하고 아름다우나, 한국이 지난 20년 크게 발전하는 동안 미국은 발전보다 성숙해지고 있다고 느낌

### 미국에서의 아이들
- 한국 훈련 방식을 버리고 미국 훈련 방식 적응 (많이 힘들었을 것) → 잘 적응해 2년간 많은 메달 + 미국 8위
- 계속 이어간다면 AI 시대에 사람이 가장 집중할 건강·운동을 더 잘 이해하는 사람으로 성장 가능
- 목표: 아이들이 이런 삶을 기록하고 성장하도록 지원, AI 기반 앱·서비스를 개인 활용을 넘어 비즈니스로 성장
